# 工单状态机

## 1. 状态定义

| 状态 | 枚举值 | 说明 |
|------|--------|------|
| 待处理 | `pending` | 新建工单的初始状态，等待施工方处理 |
| 已完成 | `completed` | 施工方完成整改，工单关闭 |
| 已拒绝 | `rejected` | 施工方拒绝处理，工单关闭 |

---

## 2. 状态流转图

```
                     [新建工单]
                         │
                    ┌────▼────┐
                    │  待处理  │◄──────────────────┐
                    └────┬────┘                    │
                         │                         │
           ┌─────────────┼─────────────┐           │
           │             │             │           │
      [解决(责任人)]  [拒绝(责任人)]  [指派他人(责任人)]
           │             │             │
      ┌────▼────┐   ┌────▼────┐   责任人变更
      │  已完成  │   │  已拒绝  │   状态不变
      └────┬────┘   └────┬────┘
           │             │
    [重新打开(发起人)]    [重新打开(发起人)]
           └──────────────┘
```

---

## 3. 操作权限详细说明

> **管理员拥有所有操作权限**，可跨越角色限制执行任意操作。

| 操作 | 允许触发角色 | 前置状态要求 | 操作说明是否必填 |
|------|------------|-------------|----------------|
| 创建工单 | 质检员（通过 Agent） | — | — |
| 解决 | 当前责任人 | 待处理 | 否 |
| 拒绝 | 当前责任人 | 待处理 | **是** |
| 指派他人 | 当前责任人 | 待处理 | **是** |
| 重新打开 | 发起人 | 已完成 / 已拒绝 | **是** |
| 编辑 | 发起人 或 当前责任人 | 任意状态 | 否 |

---

## 4. 各操作的业务逻辑

### 4.1 创建工单
- 触发方式：质检员通过 Agent 对话，Agent 返回 `create_ticket` tool_call，前端渲染确认卡片
- 创建时必须选择初始责任人（从当前项目的施工方用户中选择，必填）
- 创建成功后自动写入一条 `action=创建` 的变更记录

### 4.2 解决
- 责任人点击「解决」按钮，弹出表单
- **必填字段：** `root_cause`（问题归因）；`prevention`（预防建议）选填
- 提交后工单状态变为「已完成」
- 自动写入一条 `action=解决` 的变更记录（无需操作说明）

### 4.3 拒绝
- 责任人点击「拒绝」按钮，弹出输入框
- **必填：** 拒绝原因（写入变更记录的 `note` 字段）
- 提交后工单状态变为「已拒绝」

### 4.4 指派他人
- 当前责任人点击「指派他人」，弹出人员选择器 + 原因输入
- 只能选择**当前项目的其他施工方用户**（不能选自己）
- **必填：** 指派原因（写入变更记录的 `note` 字段）
- 提交后：工单状态保持「待处理」，`assignee_id` 更新为新责任人

### 4.5 重新打开
- 发起人点击「重新打开」，弹出输入框
- **必填：** 打开原因（写入变更记录的 `note` 字段）
- 提交后：工单状态变回「待处理」，`assignee_id` 保持关闭前的施工方不变（不清除）

### 4.6 编辑
- 发起人或当前责任人点击「编辑」，工单详情组件切换到编辑状态
- 编辑并保存后，工单状态**不变**
- 自动写入一条 `action=编辑` 的变更记录，`field_diff` 记录所有变更字段的新值

---

## 5. 变更记录规则

**每次操作（包括编辑保存）都必须创建一条 `ticket_logs` 记录，包含：**

| 字段 | 内容 |
|------|------|
| `ticket_id` | 被操作的工单 ID |
| `operator_id` | 当前操作人 |
| `action` | 操作类型枚举 |
| `field_diff` | 本次更新的字段新值（JSONB） |
| `note` | 操作说明（部分操作必填，空则存 null） |
| `created_at` | 操作时间 |
