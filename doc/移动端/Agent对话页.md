# 移动端 — Agent 对话页

**路由：** `/mobile/assistant`

> 本文档只描述页面本身的 UI 结构与交互逻辑。Agent 的完整实现（意图路由、工具调用、子 Agent 等）见：
> → [Agent模块/整体设计.md](../Agent模块/整体设计.md)

---

## 1. 页面概述

以对话形式为核心入口，支持知识问答、工单查询、创建工单三种功能，通过 Agent 自动识别用户意图并响应。

---

## 2. 页面结构

```
┌─────────────────────────┐
│   顶栏（智能助手）        │
├─────────────────────────┤
│                         │
│      消息流区域           │
│  （对话气泡 / 工单卡片）   │
│                         │
├─────────────────────────┤
│  [+] [文本输入框] [发送]   │
└─────────────────────────┘
```

---

## 3. 输入区

### 3.1 多模态输入（+ 按钮）
- 点击「+」弹出面板，提供「拍照」和「从相册选择」两个选项
- 图片上传后返回 URL，作为上下文随下一条消息一起发送给 Agent

### 3.2 文本输入框
- 标准键盘文本输入
- 支持多行（换行键发送或换行，按具体设计决定）

### 3.3 发送按钮
- 点击发送当前输入内容（文本 + 已选图片）
- **Agent 响应期间禁用输入：** 连接到后端 Agent stream 后，在 Agent 结束响应前，输入区整体不可操作（禁止发送新消息）

---

## 4. 消息流区域

消息流从上到下时间顺序排列，新消息在底部，自动滚动到底部。

### 4.1 用户消息气泡
- 文本内容（支持图片缩略图内联展示）
- 右侧对齐，蓝色背景

### 4.2 Agent 加载状态
- 发送后立即显示加载态气泡（如三个跳动的点）
- 连接 SSE stream 后实时展示 Agent 操作流程（如「正在查询知识库...」「正在分析问题...」）

### 4.3 知识问答气泡
- **触发条件：** Agent 返回文本内容
- **展示：** Markdown 渲染，支持标题、加粗、列表、代码块等
- 左侧对齐，白色/浅灰背景

### 4.4 工单创建卡片
- **触发条件：** Agent 返回 `create_ticket` tool_call
- **展示：** 不显示纯文本，渲染工单创建表单卡片
  - 预填 Agent 提取的字段（描述、位置、严重程度、专业类型等）
  - 用户可修改任何字段
  - 额外必填项：「责任人」下拉选择（从当前项目施工方用户中选择）
  - 底部「确认提交」按钮
- **提交流程：**
  1. 用户点击「确认提交」
  2. 前端调用 `POST /api/tickets` 创建工单
  3. 成功后自动向 Agent 发送包含工单 ID 的结构化 JSON（不渲染成气泡）
  4. 上一条消息中的工单卡片切换为**只读且无按钮**的展示版本
  5. Agent 回复「工单已创建成功，编号为 #XXX」等确认消息

---

## 5. 对话历史

- 按一问一答算，最多保留 6 轮历史（超出则从最旧的一轮开始丢弃）
- 页面首次加载时展示欢迎语（无历史记录时）

---

## 6. 错误处理

- 网络断开或 Agent 响应超时：展示错误提示气泡，「重新发送」按钮
- 工单创建失败：在卡片底部展示错误信息，保持卡片可编辑状态
