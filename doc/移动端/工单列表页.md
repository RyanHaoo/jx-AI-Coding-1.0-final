# 移动端 — 工单列表页

**路由：** `/mobile/tickets`

---

## 1. 页面概述

展示当前用户（当前身份所在项目）相关的工单列表，支持状态筛选、责任人筛选和排序。

> Demo 不做分页，直接 limit 50 拉取。

---

## 2. 顶部 Tabs（状态筛选）

| Tab | 对应工单状态 | 附加说明 |
|-----|-----------|---------|
| 待处理 | `pending` | Tab 右上角显示红点（数量角标） |
| 已结束 | `completed` + `rejected` | — |
| 全部 | 所有状态 | — |

---

## 3. 筛选 & 排序栏

**按责任人筛选（下拉或切换按钮）：**

| 选项 | 说明 |
|------|------|
| 我负责的 | `assignee_id = 当前用户 ID` |
| 我提出的 | `creator_id = 当前用户 ID` |
| 全部 | 不过滤 |

**排序：**

| 选项 | 说明 |
|------|------|
| 最新优先（默认） | 按 `created_at` 倒序 |
| 最旧优先 | 按 `created_at` 正序 |

**紧急置顶：** 无论排序方式，`severity = 紧急` 的工单始终置顶显示，并加**红色边框**高亮。

---

## 4. 工单列表项

每个列表条目展示以下信息：

| 元素 | 内容 |
|------|------|
| 工单编号 | `#ID` |
| 工单状态 | Badge：🔴 待处理 / 🟢 已完成 / ⚪ 已拒绝 |
| 严重程度 | Badge（紧急时红色高亮） |
| 问题描述 | 截断文本（前 20 字...） |
| 详细位置 | 小字展示 |
| 创建时间 | YYYY-MM-DD HH:mm |
| 当前责任人 | 用户小组件 |

**点击列表项：** 跳转到工单详情页 `/mobile/tickets/:id`

---

## 5. 空状态

当前筛选条件下无工单时，展示空状态插图 + 提示文字（如"暂无待处理工单"）。
