# 建筑施工质检情报员 — 项目总览

## 1. 项目简介

**建筑施工质检情报员**是一个面向建筑施工场景的 AI 驱动工单管理系统。核心定位是将现场质检过程中"发现问题 → 指派处理 → 沉淀知识"的完整链路数字化和智能化，让质检员、施工方和管理员在同一个平台上高效协作。

---

## 2. 用户角色与功能总览

| 角色 | 适用端 | 核心职责 |
|------|--------|----------|
| 质检员 | 移动端 | 发现问题，通过语音/图片发起报事；查询工单进度；查询施工规范知识 |
| 施工方 | 移动端 | 接收指派工单；反馈处理结果（解决/拒绝/指派他人） |
| 管理员 | PC 端 | 查看全量工单，进行人工干预（强制指派/重新开启）；知识运营 |
| 所有角色 | 移动端 | 通过对话框查询知识库 |
| 所有角色 | 所有端 | 登录与身份管理 |

---

## 3. 页面路由结构

```
首页 /               ：移动端入口 + PC后台入口两个大按钮
登录 /login          ：统一登录页（支持 url 参数回跳）

移动端 /mobile/
  ├─ 智能助手 /mobile/assistant   ：Agent 对话界面（知识检索 + 创建工单）
  ├─ 工单列表 /mobile/tickets     ：工单 Tab 列表
  └─ 工单详情 /mobile/tickets/:id ：工单详情页（使用工单详情组件）

PC端 /dashboard/
  ├─ 数据大盘 /dashboard/overview    ：统计概览
  ├─ 工单中心 /dashboard/tickets     ：工单列表 + 右侧详情抽屉
  └─ 知识运营 /dashboard/knowledge   ：候选池 + QA 对列表
```

---

## 4. 技术栈总览

| 层次 | 技术选型 | 说明 |
|------|----------|------|
| 前后端框架 | Next.js（统一） | 移动端、PC端、后端位于同一代码库，共用 API |
| 数据库 | Supabase | 提供数据库、RLS、Auth 全套方案 |
| AI Agent | LangChain | `createAgent` 实现核心 Agent，LangSmith 监控 |
| 知识库 & 子Agent | 扣子（Coze）平台 | 知识库托管 + 知识获取子Agent |
| AI 知识总结 | Kimi API | 简单单轮调用，用于工单 → QA 对总结 |
| 前端组件库 | shadcn/ui + lucide-react | 简约专业白底浅蓝风格 |
| UI 原型 | Google Stitch | 界面原型开发 |
| 部署 | Vercel | 一键部署 |
| 前端流式响应 | LangChain `useStream` | 处理 Agent SSE stream |

**设计风格：**
- tailwind + shadcn/ui + lucide-react
- 简约专业，白底 + 浅蓝，类似 Notion 风格
- 不用渐变、阴影堆叠、发光效果
- 全部界面使用中文
