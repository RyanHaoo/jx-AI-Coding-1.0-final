# PC 端 — 知识运营页

**路由：** `/dashboard/knowledge`

---

## 1. 页面概述

「人机协同」的核心模块，负责将已完成的优质工单通过 AI 辅助转化为问答对，并导出至扣子知识库。

> **数据一致性说明：** 「保存」操作仅将 QA 对暂存在前端状态中，工单的 `knowledge_base` 字段不会立即更新。用户刷新或关闭页面将丢失已生成的 QA 对（候选池会重新加载这些工单）。只有点击「导出 CSV → 我已完成」后才会写入数据库。Demo 阶段不实现草稿持久化。

## 3. 候选池（左侧）

### 3.1 数据源逻辑

自动拉取满足以下**全部条件**的工单：
1. `status = 已完成`
2. `prevention`（预防建议）字段不为空
3. `knowledge_base = false`（尚未被转化）

### 3.2 列表展示

使用与「工单中心」相同的列表样式，但：
- 隐藏「状态」列（已知全为已完成）
- 隐藏「操作」列
- 最左侧新增「转化」按钮列

| 列名 | 说明 |
|------|------|
| 转化 | 按钮，点击后打开转化弹窗 |
| 工单编号 | `#ID` |
| 严重程度 | — |
| 项目 | 项目小组件 |
| 专业类型 | — |
| 问题描述 | 截断 20 字... |
| 创建时间 | — |
| 发起人 | 用户小组件 |

## 4. 转化弹窗

### 4.1 布局

```
┌──────────────────────────────────────────────────────┐
│                    转化弹窗                           │
├───────────────────┬────────┬──────────────────────── ┤
│                   │        │                         │
│  工单详情组件      │  [→]   │  QA 对表单              │
│  （只读，无按钮）   │ AI总结 │  问题: [输入框]          │
│                   │        │  回答: [输入框]          │
│                   │        │                         │
│                   │        │           [保存]        │
└───────────────────┴────────┴─────────────────────────┘
```

### 4.2 左侧：工单详情

使用工单详情组件，**展示状态 + 无操作按钮**（`hideActions=true`）。

### 4.3 中间：AI 总结按钮

- 图标：右箭头（→）
- 文字：「AI 总结」
- 点击后：调用后端 AI 总结接口（Kimi API 单轮调用），将工单内容自动总结为 QA 对，填入右侧表单
- 加载期间显示 loading 状态，禁止重复点击

### 4.4 右侧：QA 对表单

| 字段 | 控件 | 是否必填 |
|------|------|---------|
| 问题 | 文本输入框 | 是 |
| 回答 | 多行文本输入框 | 是 |

底部「保存」按钮，点击后：
1. 关闭弹窗
2. 将该 QA 对添加到右侧「清洗好的 QA 对列表」
3. 从候选池**移除**该工单（纯前端状态变更，不写数据库）

---

## 5. 清洗好的 QA 对列表（右侧）

### 5.1 列表条目

每条 QA 对展示两行：
- 第一行：问题（文字截断，不换行）
- 第二行：回答（文字截断，不换行）
- 最右侧操作区：
  - 「编辑」按钮：打开转化弹窗，预填当前 QA 内容
  - 「删除」按钮：删除该条，并将对应工单重新放回候选池（纯前端）

### 5.2 导出 CSV 按钮

位于列表底部，大按钮样式。点击后执行以下流程：

**步骤 1：生成并下载 CSV 文件**
- 表头：`question,answer`
- 内容：当前所有 QA 对
- 在前端直接生成并触发下载

**步骤 2：弹出确认对话框**
- 内容：「请前往扣子知识库上传该文件」
- 按钮：「我已完成」 / 「取消」

**步骤 3A：点击「我已完成」**
1. 将这些 QA 对对应的工单 `knowledge_base` 字段更新为 `true`（调用后端 API）
2. 清空右侧 QA 对列表
3. 刷新候选池

**步骤 3B：点击「取消」**
1. 仅关闭对话框
2. 数据和界面不改变
