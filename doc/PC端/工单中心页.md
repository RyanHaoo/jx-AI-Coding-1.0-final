# PC 端 — 工单中心页

**路由：** `/dashboard/tickets`

---

## 1. 页面概述

管理员的工单流转监控中心，提供高级筛选 + 数据表格 + 右侧详情抽屉三层结构。

---

## 2. 页面布局

```
┌────────────────────────────────────────────────────────┐
│  高级筛选栏（Filter Bar）                                │
├────────────────────────────────────────────────────────┤
│                                                        │
│               数据表格（Data Table）                    │
│                                                        │
└────────────────────────────────────────────────────────┘
                                          ↑ 点击行后右侧滑出：
                                     ┌───────────────────────┐
                                     │   工单详情抽屉          │
                                     │  （Detail Drawer）     │
                                     └───────────────────────┘
```

---

## 3. 高级筛选栏

| 筛选项 | 控件类型 | 说明 |
|--------|---------|------|
| 项目名称 | 下拉单选 | 从所有项目中选择 |
| 工单状态 | 多选 | 待处理 / 已完成 / 已拒绝 |
| 严重程度 | 多选 | 轻微 / 一般 / 严重 / 紧急 |
| 专业类型 | 下拉单选 | 建筑设计 / 结构 / 给排水 |
| 关键词搜索 | 文本输入 | 匹配「问题描述」原文或「工单编号」 |

所有筛选项实时过滤表格内容（或点击「筛选」按钮触发，后期确认）。

---

## 4. 数据表格

| 列名 | 字段 | 展示样式 |
|------|------|---------|
| 工单编号 | `id` | `#ID` 格式，可点击触发详情抽屉 |
| 状态 | `status` | Badge：🔴 待处理 / 🟢 已完成 / ⚪ 已拒绝 |
| 严重程度 | `severity` | 文本；若为「紧急」整行浅红背景高亮 |
| 项目 | `project_id` | 项目小组件（`项目名称(客户名称)`） |
| 专业类型 | `specialty_type` | 文本 |
| 问题描述 | `description` | 截断前 20 字... |
| 创建时间 | `created_at` | YYYY-MM-DD HH:mm |
| 当前责任人 | `assignee_id` | 用户小组件 |
| 发起人 | `creator_id` | 用户小组件 |
| 操作 | — | 「查看详情」按钮，触发详情抽屉 |

**紧急高亮：** `severity = 紧急` 时，整行显示浅红色背景。

---

## 5. 工单详情抽屉（Detail Drawer）

点击表格行或「查看详情」按钮后，从右侧滑出抽屉（宽度约 480px），不覆盖整个页面。

**内容：** 工单详情组件，展示状态，管理员可使用所有操作按钮。

详见：→ [核心组件/工单组件.md](../核心组件/工单组件.md)

**关闭方式：** 点击抽屉外区域 / 点击关闭按钮。
