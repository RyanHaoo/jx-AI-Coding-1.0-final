# Agent 模块 — 知识查询子 Agent 说明

> 知识获取子 Agent 部署在扣子（Coze）平台，主 Agent 通过外部 API 调用获取知识文本。具体 API 调用方式和接口定义后期补充。

---

## 1. 设计定位

**职责：** 在多种知识源中查询输入的主题，返回详细的知识文本。

**不负责：** 意图路由、用户上下文、外部对话，只专注搜索与检索。

---

## 2. 输入规范（主 Agent → 子 Agent）

主 Agent 传入的内容是**精练的待查询主题**，而非用户原始提问。

---

## 3. 输出规范（子 Agent → 主 Agent）

子 Agent 返回详细的知识文本（Markdown 格式），包含：
- 相关规范条文（如 GB 标准编号 + 条文内容）
- 具体数值/要求
- 来源说明

主 Agent 接收后，整合为用户友好的回答

---

## 4. 调用接口

> 🚧 **待补充：** 具体 API URL、请求格式、认证方式、响应格式等，后期在扣子平台配置完成后补充至此。

---

## 5. 知识源

子 Agent 在扣子平台上接入以下知识源（后期可扩展）：
- 扣子知识库（由知识运营模块持续维护的 QA 对）
- 建筑施工规范文档（国家/行业标准）
- 其他在线资源（扣子内置搜索插件）
