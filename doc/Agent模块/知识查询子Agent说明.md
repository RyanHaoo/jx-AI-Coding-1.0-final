# Agent 模块 — 知识查询子 Agent 说明

> 知识获取子 Agent 部署在扣子（Coze）平台，主 Agent 通过外部 API 调用获取知识文本。具体 API 调用方式和接口定义后期补充。

---

## 1. 设计定位

**职责：** 在多种知识源中查询输入的主题，返回详细的知识文本。

**不负责：** 意图路由、用户上下文、外部对话，只专注搜索与检索。

**多 Agent 分工价值：**
- 主 Agent 只关心意图路由和工具调度，把问题丢给子 Agent 即可获取知识
- 子 Agent 只关心搜索，不受复杂外部上下文干扰
- 隔离上下文 → 各自精专 → 更高准确度和稳定性
- 子流程拆小更容易独立调试优化

---

## 2. 输入规范（主 Agent → 子 Agent）

主 Agent 传入的内容是**精练的待查询主题**，而非用户原始提问。

主 Agent 内部通过 Query Rewriter 对用户提问进行改写后再传入：

### 2.1 Query Rewriter 规则

| 规则 | 说明 |
|------|------|
| 指代消解 | 将代词（它/那个/这个）替换为具体实体名词 |
| 补全上下文 | 从前序对话中提取省略的背景信息（系统名称、项目、前置条件） |
| 意图提纯 | 去除口语化前缀和语气词（请问/帮我查一下），保留核心检索词 |
| 不回答 | 只重构提问，不推测答案 |
| 去重 | 前序会话中已检索清楚的主题不再重复传入 |

**多议题格式：** 使用两个换行隔开

**示例：**
- 用户提问：「做学校设计的时候，该怎么设计门框和窗宽？要考虑哪些采光因素？」
- 传入子 Agent：
  ```
  学校建筑设计中门框、窗宽的设计尺寸要求，门窗数量设置原则

  学校建筑设计中门窗设置对采光的影响
  ```

### 2.2 输入格式

```
{改写后的查询主题1}

{改写后的查询主题2（若有）}
```

---

## 3. 输出规范（子 Agent → 主 Agent）

子 Agent 返回详细的知识文本（Markdown 格式），包含：
- 相关规范条文（如 GB 标准编号 + 条文内容）
- 具体数值/要求
- 来源说明

主 Agent 接收后，整合为用户友好的回答，以 Markdown 气泡渲染展示。

## 5. 知识源

子 Agent 在扣子平台上接入以下知识源（后期可扩展）：
- 扣子知识库（由知识运营模块持续维护的 QA 对）
- 建筑施工规范文档（国家/行业标准）
- 其他在线资源（扣子内置搜索插件）
