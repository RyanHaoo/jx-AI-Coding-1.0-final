# Agent 模块 — 整体设计

---

## 1. 架构总览

```
用户（多角色）
    ↕ 多模态输入（文本 + 图片）
前端 UI（/mobile/assistant）
    ↕ SSE Stream（LangChain useStream）
主 Agent（Next.js 后端）
    ├── 知识获取子 Agent（外部 API，扣子平台）
    ├── 工单查询 MCP（stdio，本地启动）
    └── create_ticket tool_call（HITL，通知前端渲染卡片）
```

**关键设计要点：**
- **单一身份：** 用户登录时固定单一身份（项目 + 角色），整个对话期间不变
- **多模态：** 用户可带图片输入，主 Agent 将图片 URL 纳入上下文
- **主 Agent 是唯一对话出口：** 子 Agent 不直接与用户对话，只返回知识文本给主 Agent
- **知识子 Agent 多轮检索：** 信息不足时自动追加查询，不是一次性返回
- **HITL 工单创建：** Agent 不直接创建工单，通过 `create_ticket` tool_call 通知前端，由用户确认后调用 API 创建

---

## 2. 主 Agent

**开发框架：** LangChain `createAgent`（ReAct 模式），LangSmith 监控

**职责：** 接收用户输入 → 识别意图 → 调度工具/子 Agent → 流式返回响应

### 2.1 意图路由规则

| 意图 | 判断依据 | 路由 |
|------|----------|------|
| 知识问答 | 用户询问施工规范、工艺标准、设计要求等知识性问题 | 知识获取子 Agent API |
| 工单查询 | 用户询问工单进度/状态，或提到工单编号 | queryTicket MCP |
| 创建工单 | 用户描述现场质量问题，要求报事 | `create_ticket` tool_call |
| 混合意图 | 同时涉及知识查询 + 报事 | 先知识查询，再根据追问决定是否创建工单 |
| 超范围 | 与建筑施工质检无关 | 礼貌告知超出范围 |

### 2.2 上下文管理

- **对话历史：** 按一问一答计算，最多保留 6 轮（超出时丢弃最旧一轮）
- **当前用户信息：** 通过 System Prompt 注入（姓名、部门、角色、项目）
- **图片上下文：** 用户上传图片后，URL 随对话一起传递

### 2.3 内部功能模块

| 模块 | 职责 |
|------|------|
| 意图识别 | 解析用户输入，按规则路由 |
| 多轮对话管理 | 维护对话历史，管理追问与上下文 |
| 工具调度 | 调用 queryTicket MCP 和 create_ticket tool_call |
| 流式消息广播 | 向前端推送 SSE stream |
| 外部知识接入 | 调用知识获取子 Agent API |
| Query Rewriter | 将用户原始提问改写为标准检索查询 |

## 5. 典型用户流程

### 流程 A：知识问答
```
用户提问 → 主Agent意图识别 → Query Rewriter改写 → 知识子Agent检索 → 主Agent整合回答
```

### 流程 B：工单查询
```
用户提问 → 主Agent意图识别 → 调用queryTicket MCP → 主Agent汇报进度
```

### 流程 C：创建工单
```
用户描述问题 → 主Agent识别报事参数 → create_ticket tool_call
→ 前端渲染创建卡片（含责任人选择）→ 用户确认提交
→ 前端调用 POST /api/tickets → 返回ticket_id给Agent → Agent告知创建完成
```

### 流程 D：知识 + 工单联动
```
用户上传施工图截图 → 主Agent多模态理解 → 知识子Agent检索规范 → 主Agent回答判断

用户追问"帮我建单" → 主Agent整理上下文 → create_ticket tool_call → 前端确认
```
