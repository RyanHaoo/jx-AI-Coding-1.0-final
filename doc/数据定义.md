# 数据定义

> 本项目为演示 Demo，严格按以下字段进行开发，不做扩展。

---

## 1. 实体列表

| 实体 | 表名（建议） | 说明 |
|------|-------------|------|
| 用户 | `profiles` | 与 Supabase Auth 关联，存储业务信息 |
| 项目 | `projects` | 施工项目 |
| 用户身份 | `user_roles` | 用户 × 项目 × 角色三元组 |
| 工单 | `tickets` | 核心业务实体 |
| 工单变更记录 | `ticket_logs` | 工单全生命周期操作日志 |

---

## 2. 用户角色定义

系统中有三种角色，通过「用户身份」表与项目绑定：

| 角色 | 说明 | 所在端 |
|------|------|--------|
| 质检员 | 发现问题、发起报事 | 移动端 |
| 施工方 | 接收并处理工单 | 移动端 |
| 管理员 | 全局监控与干预，知识运营 | PC 端 |

> **重要约束：** 用户登录后必须选择一个「项目 + 角色」组合作为本次会话身份，在整个会话期间固定，不支持运行中切换。一个用户可以在不同项目中担任不同角色。

---

## 3. 数据库表定义

### 3.1 用户 `profiles`

> 鉴权由 Supabase Auth 负责，此表只存储业务信息。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | UUID | 主键，对应 Supabase Auth 用户 ID |
| `number` | string | 用户工号，不可重复不可编辑，用于登录 |
| `name` | String | 用户姓名 |
| `department` | String | 所在部门 |
| `avatar_url` | URL | 头像图片链接 |

**权限：** 仅管理员可写；所有登录用户可读。

---

### 3.2 项目 `projects`

| 字段名 | 类型 | 枚举值/说明 |
|--------|------|-------------|
| `id` | 自增 INT | 主键 |
| `name` | String | 项目名称 |
| `city` | String | 项目城市 |
| `client_name` | String | 客户公司名称 |
| `type` | Enum | 地产项目 / 园区项目 / 景观项目 / 居住区项目 / 政府项目 |

**权限：** 仅管理员可写；所有登录用户可读。

---

### 3.3 用户身份 `user_roles`

> 「用户」到「项目」的多对多关联表，同时记录该关联下的角色。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | 自增 INT | 主键 |
| `user_id` | 用户外键 | 关联 `profiles.id` |
| `project_id` | 项目外键 | 关联 `projects.id` |
| `role` | Enum | 质检员 / 施工方 / 管理员 |

**权限：** 仅管理员可写；所有登录用户可读。

---

### 3.4 工单 `tickets`

> **写入权限说明：** 工单只能通过 Next.js 后端 API 写入，不允许直接调用 Supabase Auto API 写操作。原因是工单变更涉及复杂状态流转校验和自动创建变更记录，需要在后端统一处理。

| 字段名 | 类型 | 枚举值/说明 |
|--------|------|-------------|
| `id` | 自增 INT | 主键（工单编号） |
| `status` | Enum | 待处理（默认）/ 已完成 / 已拒绝 |
| `severity` | Enum | 轻微 / 一般 / 严重 / 紧急 |
| `created_at` | DateTime | 创建时间 |
| `creator_id` | 用户外键 | 发起人（质检员） |
| `project_id` | 项目外键 | 所属项目 |
| `assignee_id` | 用户外键 | 当前责任人（施工方）；创建时强制选择 |
| `specialty_type` | Enum | 建筑设计专业 / 结构专业 / 给排水专业 |
| `description` | String | 问题描述 |
| `location` | String | 详细位置（如：东区一期-3#住宅楼-标准层） |
| `images` | Array\<URL\> | 图片列表，支持多张 JPG，非必填 |
| `detail` | Text | 问题详情，非必填 |
| `root_cause` | Text | 问题归因，解决时必填 |
| `prevention` | Text | 预防建议，非必填 |
| `knowledge_base` | Boolean | 是否已转化为知识库，默认 false |

**权限：** 登录用户只可读取当前身份所在项目下的条目；写操作统一走后端 API。

**枚举参考值：**
- 问题描述参考（建筑）：尺寸偏差、标高不符、节点做法未落地、防火分区违规
- 问题描述参考（结构）：构件配筋不足/错位、混凝土强度不合格、钢筋保护层偏差
- 问题描述参考（给排水）：管道材质/规格不符、接口渗漏、坡度不合理、通水/试压不达标

---

### 3.5 工单变更记录 `ticket_logs`

> 记录工单的每一次状态变更或内容编辑，确保全生命周期可追溯。**写入权限同工单，只能通过后端写入。**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | 自增 INT | 主键 |
| `ticket_id` | 工单外键 | 关联 `tickets.id` |
| `operator_id` | 用户外键 | 操作人 |
| `action` | Enum | 创建 / 解决 / 拒绝 / 指派他人 / 重新打开 / 编辑 |
| `field_diff` | JSONB | 本次操作更新的字段 key-value（只存新值） |
| `note` | Text | 操作说明（部分操作必填，见工单状态机） |
| `created_at` | DateTime | 操作时间 |

**权限：** 登录用户只可读取当前身份所在项目下的工单相关条目。
