# 核心组件 — 工单详情组件

> 双端共用组件，同时用于移动端（工单详情页）和 PC 端（工单中心抽屉、知识运营转化弹窗）。

---

## 1. 组件概述

工单详情组件有两个状态，通过 `mode` prop 控制：

| 状态 | 说明 |
|------|------|
| `view`（展示） | 只读展示工单信息，底部显示操作按钮 |
| `edit`（编辑） | 部分字段可编辑，底部显示"保存"按钮 |

---

## 2. 信息分组与字段

### 2.1 基础信息

| 字段 | 编辑状态 | 展示状态 |
|------|---------|---------|
| 工单 ID | 只读 | 卡片展示 |
| 工单状态 | 只读 | Badge 标签 |
| 创建时间 | 只读 | 格式化时间 |
| 发起人 | 只读 | 用户小组件 |
| 所在项目 | 只读 | 项目小组件 |
| 当前责任人 | 只读 | 用户小组件 |

> 展示状态下，若有图片则展示首图缩略图预览。

### 2.2 工单详情（可读写）

| 字段 | 类型 | 是否必填 |
|------|------|---------|
| 严重程度 | Enum 选择 | 是 |
| 专业类型 | Enum 选择 | 是 |
| 问题描述 | 文本输入 | 是 |
| 详细位置 | 文本输入 | 是 |
| 图片 | 多图上传 | 否 |
| 问题详情 | 长文本 | 否 |

### 2.3 工单复盘（可读写）

| 字段 | 类型 | 说明 |
|------|------|------|
| 问题归因 | 长文本 | 解决时必填 |
| 预防建议 | 长文本 | 选填 |
| 是否转化知识库 | 只读 Boolean | 不可编辑，仅展示 |

---

## 3. 底部操作区

### 3.1 编辑状态

底部固定显示「保存」按钮。保存成功后：
1. 组件切换回展示状态
2. 自动写入一条 `action=编辑` 的变更记录

### 3.2 展示状态 — 操作按钮

按钮根据**当前用户身份**和**工单状态**动态显示/隐藏：

| 操作按钮 | 显示条件 | 点击交互 |
|---------|---------|---------|
| 编辑 | 当前用户是发起人 或 当前责任人 | 切换组件为编辑状态 |
| 解决 | 当前用户是当前责任人；工单状态=待处理 | 弹出表单（必填问题归因） |
| 拒绝 | 当前用户是当前责任人；工单状态=待处理 | 弹出输入框（必填拒绝原因） |
| 指派他人 | 当前用户是当前责任人；工单状态=待处理 | 弹出人员选择器 + 原因输入 |
| 重新打开 | 当前用户是发起人；工单状态=已完成 或 已拒绝 | 弹出输入框（必填打开原因） |

> **管理员：** 拥有所有操作权限，以上所有按钮在满足状态前置条件时均显示。

---

## 4. 无操作按钮模式

当组件用于知识运营转化弹窗左侧时，强制为展示状态且不显示任何操作按钮（通过 `hideActions` prop 控制）。

---

## 5. Props 接口（参考）

```typescript
interface TicketDetailProps {
  ticketId: number;
  mode?: 'view' | 'edit';           // 默认 'view'
  hideActions?: boolean;             // 隐藏所有操作按钮，默认 false
  onSave?: () => void;              // 保存成功回调
  onAction?: (action: string) => void; // 操作完成回调
}
```
